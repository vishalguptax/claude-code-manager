# v2.5.0

_Released 2026-07-05_

The Account tab now tracks live data — models, pricing, quota, and usage stay in sync with what Claude Code actually reports — alongside a reworked statusline installer, broad UI/accessibility fixes, and a snappier, keep-alive tab shell.

## ✨ Features

- account: live-data sync, statusline rework, perf ([f83b910](https://github.com/vishalguptax/claude-code-manager/commit/f83b9107b9dc1deb8547e1482a2a5e9813e26b09))
  - Model dropdown discovers any `claude-{family}-{version}` from the CLI binary (new families like Fable appear without an extension release) and re-scans when the binary changes.
  - Pricing refreshed to July 2026 rates; usage cost is computed against the correct per-model tiers.
  - Statusline install is now global-first with per-workspace local overrides — it never writes machine-specific paths into shared project settings, repairs entries poisoned by earlier versions, and self-heals across CLI and extension updates.
  - Session-aware statusline cache: "Default (model)" only names a model when concurrent sessions agree, otherwise shows "auto".
  - Persistent daily usage rollup survives Claude Code's transcript cleanup, so long-term history isn't lost.
  - Host responsiveness: keychain-read caching, batched watcher re-parses, deduped account pushes, a busy-bar for slow round-trips, and slow-handler tripwires that name the culprit in the Extension Host log.

## ⚡ Performance

- webview: keep tabs mounted instead of remounting on switch ([e11fecc](https://github.com/vishalguptax/claude-code-manager/commit/e11fecc85b48ea34e8588932e0703aef2d74d475)) — revisiting a tab is now instant (no re-import, re-request, or DOM rebuild); only the first visit pays the load cost.
