# v1.11.5

_Released 2026-06-01_

Self-heal the statusline tap on activation. If the project / local `.claude/settings.json` that held our tap line gets reverted (a teammate's PR, a `git checkout`, a merge wipe), the sidecar still recorded "installed" but Claude Code was running a stale (or no) statusline → the Quota card froze even though Enable had been clicked. Now the extension re-installs silently at the effective scope on every activation so the user never has to click Enable twice.

## 🐛 Bug Fixes

- account: re-install the statusline tap silently when the sidecar says it's installed but the effective scope no longer holds our tap. No-op when nothing to heal (single fs read).
