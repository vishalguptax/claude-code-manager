# v2.5.2

_Released 2026-07-06_

Fixes a regression where the Account and Config tabs could hang on a skeleton for 10-20 seconds, and removes the stalls that made account switching feel slow.

## 🐛 Bug Fixes

- account: restore instant Account/Config load, cut switch stalls ([7244d81](https://github.com/vishalguptax/claude-code-manager/commit/7244d81529a5844b78dd82ae58be820bbc79bc2d))
  - The Account and Config tabs both request the same account payload and, with keep-alive tabs, share one long-lived webview — so the payload-dedupe added in 2.5.0 dropped the second tab's reply as "unchanged", leaving its skeleton stuck until the next content change (up to 10-20s during a session). Dedupe is now opt-in: replies to an explicit request always deliver; only background watcher pushes dedupe.
  - Account switching no longer clears the model cache (which forced a full ~236 MB CLI-binary re-scan on the very reparse the switch was waiting on) — the model list is read from the binary and doesn't change per account, and genuine CLI upgrades are still picked up automatically.
  - The same physical CLI binary reached via two paths is now scanned once, and a large active-session transcript yields the event loop mid-parse so it can't block the host.
  - The switch confirmation now sets the honest expectation that a Claude terminal already running keeps its account until it restarts (the credentials swap immediately for new sessions; a running CLI holds its token in memory).
