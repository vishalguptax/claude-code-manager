# v2.2.2

_Released 2026-07-01_

The session panel now tells the truth and the whole extension stops freezing: dead sessions no longer masquerade as active, newly started sessions appear on their own, a pending question shows the right colour, and no user action blocks the host on a multi-second synchronous read.

## 🐛 Bug Fixes

- **sessions: correct live-state + eliminate host-blocking on actions** ([891645f](https://github.com/vishalguptax/claude-manager/commit/891645f))
  - **Stale "Active" sessions** — liveness was decided purely by whether *some* process still owned the recorded PID, so a hard-killed session whose PID the OS had recycled showed as active forever (especially on Windows). A session is now "live" only when the OS process start time matches the CLI-recorded `startedAt` (cross-platform: `/proc` on Linux, `ps` on macOS, CIM on Windows).
  - **New sessions not appearing** — the liveness poll now self-heals, pulling in a live session that's missing from the cached list so a freshly started session surfaces within one tick even if the filesystem watcher misses its create event.
  - **Terminal mislink** — the active-session watcher rejects reused parent-shell PIDs.
  - **Green dot while waiting on you** — a pending `AskUserQuestion` / `ExitPlanMode` now shows the orange "needs you" dot even when the CLI leaves a stale `busy` status.

## ⚡ Performance

Removed every synchronous multi-second operation from user-action paths — the extension host is single-threaded, so these were freezing the whole UI (Settings tab, Export Brain, Run Diagnostics, and any click) for up to ~10s. All are now cached and warmed in the background off the event loop ([891645f](https://github.com/vishalguptax/claude-manager/commit/891645f)):

- **Model discovery** no longer reads the ~236 MB CLI binary synchronously on every settings interaction — async, warmed at activation, and memory-bounded (one binary at a time).
- **Usage aggregation** no longer re-reads the entire transcript corpus synchronously on every account push — async with event-loop yields; the render path serves a cache.
- **Diagnostics** (`claude --version`) and the process-liveness queries are async, and the `node` path lookup is memoized — no more 5s freeze on Run Diagnostics.

## 💅 UI / Styling

- Rebuilt the session-detail loading skeleton to mirror the real layout (header, stat strip, actions, message cards) so it no longer pops/reflows when content lands ([891645f](https://github.com/vishalguptax/claude-manager/commit/891645f)).
