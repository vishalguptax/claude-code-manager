# v2.0.0 (work in progress)

Per-feature migration notes for the v2 Preact revamp. Each F2 session appends
its own section; the F3 integration session assembles the final CHANGELOG.

## account

- Migrated the Account tab webview from vanilla DOM string-templates to
  Preact functional components (Profile / Quota / Usage views + leaf
  components).
- State moved to `@preact/signals`; postMessage sends go through the
  typed `useAccountApi()` hook, narrowed to the shared `WebviewMessage`
  union (compile-time validated).
- Inbound `accountData` / `quotaData` / `error` messages are validated
  via the shared valibot `parseMessage` gate (in the F1 message bus)
  before reaching the feature handler.
- Heatmap, currency/number formatting, and usage aggregation extracted
  into pure, fully-unit-tested modules.
- All feature styling consolidated in `src/styles/account.css`; no
  inline styles in TSX (dynamic geometry — heatmap grid coords and
  progress-bar widths — stays inline as it is data-driven, matching the
  F1 `VirtualList` precedent).

## hooks

- Migrated the hooks tab webview from vanilla DOM to Preact + @preact/signals.
- All webview → host messages are built through typed helpers bound to the
  shared `WebviewMessage` protocol; inbound `hooks` data is validated by the
  shared valibot message bus before reaching the feature.
- List view scope filtering, search, and per-event grouping preserved;
  lists over 50 hooks now render through the shared `<VirtualList />`.
- Detail view keeps copy / edit / toggle / delete / open-settings actions;
  plugin-sourced hooks stay read-only.
- Feature styles moved to `src/styles/hooks.css` (no inline styles, no innerHTML).
