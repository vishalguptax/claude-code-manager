# v2.0.0 (work in progress)

Per-feature migration notes for the v2 Preact revamp. Each F2 session appends
its own section; the F3 integration session assembles the final CHANGELOG.

## account

- Migrated the Account tab webview from vanilla DOM string-templates to
  Preact functional components (Profile / Quota / Usage views + leaf
  components).
- State moved to `@preact/signals`; postMessage sends go through the
  typed `useAccountApi()` hook, narrowed to the shared `WebviewMessage`
  union (compile-time validated).
- Inbound `accountData` / `quotaData` / `error` messages are validated
  via the shared valibot `parseMessage` gate (in the F1 message bus)
  before reaching the feature handler.
- Heatmap, currency/number formatting, and usage aggregation extracted
  into pure, fully-unit-tested modules.
- All feature styling consolidated in `src/styles/account.css`; no
  inline styles in TSX (dynamic geometry — heatmap grid coords and
  progress-bar widths — stays inline as it is data-driven, matching the
  F1 `VirtualList` precedent).

## hooks

- Migrated the hooks tab webview from vanilla DOM to Preact + @preact/signals.
- All webview → host messages are built through typed helpers bound to the
  shared `WebviewMessage` protocol; inbound `hooks` data is validated by the
  shared valibot message bus before reaching the feature.
- List view scope filtering, search, and per-event grouping preserved;
  lists over 50 hooks now render through the shared `<VirtualList />`.
- Detail view keeps copy / edit / toggle / delete / open-settings actions;
  plugin-sourced hooks stay read-only.
- Feature styles moved to `src/styles/hooks.css` (no inline styles, no innerHTML).
Per-feature migration notes for the Preact revamp. Assembled into the final
CHANGELOG during F3 integration.

## mcp

- Migrated the MCP servers tab to Preact with `@preact/signals` state.
- Added valibot validation on every postMessage, on both the webview send
  path (`createMcpApi`) and the host receive path (`handleMcpMessage`).
- Extracted host dispatch into `features/mcp/messageHandlers.ts` (validated,
  self-contained, no cross-feature imports).
- Large server lists (>50) now render through the shared windowed
  `<VirtualList />`; smaller lists render grouped by scope as before.
its own section; the integration session assembles the final CHANGELOG.

## commands

- Migrated the commands webview from vanilla DOM to Preact with `@preact/signals`.
- Inbound messages from the webview are now validated against the shared valibot protocol on the host (`dispatchCommandsMessage`), and host messages are routed into feature signals via the shared message bus.
- List view groups commands by scope, debounces search, and windows the rows with `<VirtualList />` once the list grows past 50 entries.
- Extracted host dispatch into `messageHandlers.ts` with injected VS Code capabilities so it is unit-testable without the extension host.
# v2.0.0 — Work in progress

Per-feature changelog entries assembled during the F2 migration. The
integration session (F3) consolidates these into the final CHANGELOG.

## skills

- Migrated the Skills tab webview from vanilla DOM to Preact + @preact/signals.
- All webview→host messages now flow through the shared valibot-validated protocol via the `useApi()` hook; inbound `skills` / `skillDetail` / `settings` messages are parsed by the shared message bus.
- List rendering switches to the shared `<VirtualList />` above 50 items; smaller lists render directly. Grouping (scope → folder) extracted into a pure, unit-tested `grouping.ts`.
- Feature styles consolidated in `src/styles/skills.css` (added a `scope-plugin` badge); no inline styles or `innerHTML` remain.
- Removed cross-feature imports — the feature no longer reaches into `sessions/webview` or the shared `marketplace` / `extensionStatus` singletons; marketplace URL and Claude Code install state are tracked in feature-local signals fed by the host `settings` message.
Per-feature changelog entries assembled during the F2 migration. Finalized by
the F3 integration session.

## agents

- Migrated the agents webview from vanilla DOM to Preact with `@preact/signals`.
- Inbound `agents`/`error` messages now flow through the shared message bus and
  are validated by valibot (`parseMessage`); outbound `getAgents`/`openAgentFile`
  go through the typed `useApi()` bridge.
- Large agent lists (> 50 entries) render through the shared windowed
  `VirtualList`; smaller lists keep scope-grouped sections (Project / Global /
  Plugin). No user-visible behavior change.
- Feature styles consolidated into `src/styles/agents.css` (no inline styles, no
  `innerHTML`).
