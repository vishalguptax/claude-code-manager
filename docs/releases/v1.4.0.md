# v1.4.0

_Released 2026-04-19_

Full-text session search, a proper branch-filter dropdown, and an account tab overhaul — the largest sessions pass since import/export.

## ✨ Features

- sessions: full-text search, branch filter dropdown, account polish [minor] ([3856fdd](https://github.com/vishalguptax/claude-code-manager/commit/3856fddbefba1a6be3992435383ffdc466e46e05))

### Sessions

- **Full-text search** — search now matches inside every message of every session transcript, not just titles and metadata. An extension-host content cache (50 KB per session, lazily built in chunked `setTimeout` ticks with a generation counter) keeps the UI responsive even on thousands of sessions.
- **Branch filter dropdown** — sits beside the project dropdown. Lists every branch that has sessions in the active project scope, counts sessions per branch, and tags the workspace's current git branch with a `current` label. Picking a different project auto-resets the branch filter so stale selections never hide the new project's sessions.
- **Live branch tracking** — the dropdown reacts to git checkouts via the VS Code Git extension's repository events, so the "current" marker always reflects where you actually are.

### Account

- **Dynamic model list** — pulls Claude model IDs straight from the installed Claude CLI (handles both the legacy JS bundle and the current native binary). No more hardcoded model list drifting out of date when Anthropic releases a new model.
- **Correct usage totals** — 7-day / 30-day / All-time filters now anchor to the latest data date and read cache totals for "All", so the numbers match what Claude itself reports.
- **Config backup restore** — if `~/.claude.json` is ever empty or corrupt, the Account tab shows a banner offering a one-click restore from the newest valid file in `~/.claude/backups/`. Keeps your account + settings through a broken write without re-logging-in.
