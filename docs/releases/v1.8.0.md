# v1.8.0

_Released 2026-05-08_

Account profile switching now survives Claude CLI's token-rotation lifecycle, sessions pick up the CLI-generated topic title, and usage stats fill in days the stats cache hasn't caught up to yet.

## ✨ Features

- sessions: read CLI ai-title entries for session display name ([fa89934](https://github.com/vishalguptax/claude-code-manager/commit/fa8993438c4f245a2e178383601d955fee227256))

## 🐛 Bug Fixes

- account: keep saved profile credentials current with token rotation — switching back to a saved account no longer 401s after the slot's snapshot has been used. `switchProfile` re-syncs the outgoing slot before swap and the credentials watcher mirrors live rotations into the matching slot continuously. Identity matching upgraded to `oauthAccount.accountUuid` (account-distinct) instead of the device-stable `userID`. ([e49a58b](https://github.com/vishalguptax/claude-code-manager/commit/e49a58bf1c76de205def837e1e14d78ab6d8c0b9))
- account: heatmap + token totals miss days past stats-cache cutoff ([5a816bc](https://github.com/vishalguptax/claude-code-manager/commit/5a816bcf09717fbd94abde425db9d75e5e7129ce))
