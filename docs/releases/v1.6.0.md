# v1.6.0

_Released 2026-04-24_

New Config tab, Brain backup / restore, and a deep rewrite of the multi-account switcher. Account tab is identity-focused now; behavior, permissions, tools, and portable brain transfer live under their own tab.

## ✨ Features

- Config tab + Brain backup + account-switch hardening [minor] ([1544cd7](https://github.com/vishalguptax/claude-code-manager/commit/1544cd7c6f0745aa8ffd78af015d7aa4ff6f35e2))

### Highlights

**New Config tab**
Settings and Permissions moved out of Account and into a dedicated Config tab. Adds `permissions.defaultMode` picker, `permissions.additionalDirectories` via native folder picker, `includeCoAuthoredBy` toggle, `spinnerTipsEnabled` toggle, `cleanupPeriodDays` numeric, and an `effortLevel` radio group mirroring the CLI's `/effort` slash command (surfaces unknown tiers verbatim so new Claude Code values land without an extension update). Permission list now has search, an inline pattern hint, and a confirm-before-remove step; Add button splits into "Add allowed" / "Add denied". Reset-settings button renames the scope's `settings.json` to a timestamped `.bak` so Claude CLI regenerates a clean one. Top-bar gear icon retired — "Extension settings" is a Config footer button.

**Brain backup / restore**
Two new commands — `Claude Manager: Export Brain…` and `Claude Manager: Import Brain…` — plus buttons in the Config tab. Packages your Claude setup (global `~/.claude/` + current workspace `.claude/`) into a single `.claudebrain.zip`. Scope picker chooses global, project, or both. Zero runtime dependency: custom STORE-only ZIP codec (~180 LOC, readable by Explorer, Finder, and any `unzip`). Importer merges `mcpServers` additively, preserves existing files as `.imported` siblings, and warns when imported hooks reference absolute paths that don't exist on the target machine. Sessions, credentials, identity, and `manager-accounts/` are never included.

**Account-switch hardening**
Three interlocking bugs traced to one root: `~/.claude.json` is lag-written by Claude CLI during `/login`, so reading identity from it during the transition mis-attributed the new account to the previous saved slot. Symptoms that all disappear in this release:
- Save button hidden after a fresh `/login` to a new account
- QuickPick switcher showed the old account as Active after login
- "Log in as new account" silently replaced the previously active saved profile
Active-profile detection now trusts the JWT access-token claims (authoritative) over `~/.claude.json` (lagging). Save-profile dedupe keys off the same JWT. `switchProfile` merges identity keys into the live `.claude.json` instead of overwriting it wholesale — your `projects`, `numStartups`, migration flags, caches, and MCP config all survive a switch now. The two-file swap is crash-safe with `.bak` rollback. The file watcher posts an info toast + Save-as-profile shortcut when a CLI-side `/login` changes identity outside the switcher. Duplicate saved slots flagged in the QuickPick when `userID + email` both match. Stale email on a saved active slot is overlaid with the live JWT email so an Anthropic-side email change shows up immediately.

**Usage heatmap**
Cell intensity scales on token count now (falls back to `messageCount` only when tokens are absent). Tooltip leads with the token total followed by message + session counts.

**Quota robustness**
Three-attempt exponential backoff (80ms → 250ms → 600ms, ~930ms window) when reading `~/.claude/.credentials.json` — clicking Refresh during token rotation no longer falsely reports "No credentials". Transient-state reads get a distinct "try again in a moment" message instead of the generic missing-credentials error. Currency values render via `Intl.NumberFormat` with ISO 4217 fraction-digit mapping — AUD users stop seeing "23346.00 AUD" for a $233.46 bill.

**Signed-out flow**
Account tab in signed-out state now surfaces a Switch button when saved profiles exist. Previously only `/login` was reachable, forcing users to lose their saved accounts on the way back in.

**Brain imports hook-safety warning**
After importing a `.claudebrain.zip`, Claude Manager scans the incoming `settings.json` hooks for absolute-path commands that don't exist on the target machine. Any misses are surfaced in a second, dismissible modal so you can fix them before Claude silently fails the hook at runtime.

### UI polish

- Brain section gets a proper package icon (was silently blank because the icon name didn't exist in the sprite).
- Inline `<code>` chips in field hints get horizontal padding so they read as chips rather than bare text.
- QuickPick rows: email de-duplicated between row 1 and row 2 (now only on row 2).
