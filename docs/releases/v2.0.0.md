# v2.0.0

_Released 2026-05-23 · first shipped as `2.0.0-beta.1`_

The webview is rebuilt from the ground up on **Preact + @preact/signals**, with
every host↔webview message validated at runtime by **valibot** and a single
shared protocol as the source of truth. This is a migration release: the UI,
features, settings keys, command IDs, and the "100% local, zero telemetry"
promise are all preserved. No user action is required to upgrade — see
[docs/migration/v1-to-v2.md](../migration/v1-to-v2.md).

## ✨ Architecture

- **Preact webview** — all seven feature tabs (Sessions, Skills, Commands,
  Hooks, MCP, Agents, Account) plus the Config tab rebuilt as functional
  components. No vanilla DOM string-templates, no `innerHTML`.
- **Signals state** — feature state moved to `@preact/signals`; views re-render
  reactively. No global store, no provider tree.
- **Validated protocol** — every postMessage shape lives in
  `src/shared/protocol/messages.ts` and is parsed by a valibot schema on both
  the send and receive paths. Shape drift fails at compile time and is rejected
  at runtime.
- **Code-split webview** — esbuild ships a tiny shell (`main.js`) and lazy-loads
  each feature as its own chunk on first tab activation.
- **Biome** replaces ESLint + Prettier; **size-limit** gates the bundle in CI.
- **Virtualized lists** — Sessions, and any feature list over ~50 rows, render
  through a shared windowed `<VirtualList />` so 5,000+ sessions scroll in
  constant time.

## Per-feature notes

### Sessions

- Virtualized list, live status dots, debounced search (250 ms),
  project/branch/date filters, bulk mode, and the windowed detail view rebuilt
  as Preact components.
- Host-side `viewProvider.ts` decomposed into focused modules (message
  handlers, live state, watchers); `parser.ts` split into meta / history /
  grouping. Search index gained LRU eviction.

### Account

- Profile / Quota / Usage rebuilt as Preact views; heatmap, currency/number
  formatting, and usage aggregation extracted into pure, unit-tested modules.

### Config

- Settings / Permissions / Settings-history / Brain-backup migrated to Preact
  and restored to the tab bar, wired to the existing host handlers through the
  shared protocol.

### Skills, Commands, Hooks, MCP, Agents

- Each migrated to Preact + signals with valibot-validated messaging and
  feature-local host dispatch (`messageHandlers.ts`). Cross-feature imports
  removed in favour of the host registry. Feature styles consolidated under
  `src/styles/{feature}.css`.

## 🔒 Security

- Branch-name validation added before any `claude --resume` shell invocation
  (resume / fork / restore-workspace), closing a shell-injection vector via
  crafted git refs.
- CSP hardened: `default-src 'none'`, nonce-only module scripts, `connect-src
  'none'`. No `unsafe-eval`.

## 🧰 Tooling

- `@vscode/test-electron` integration smoke (activation, command registration,
  view resolution) runs under xvfb in CI.
- Performance harness (`scripts/perf/`) generates a configurable N-session
  fixture and reports parse / group / search latency, memory, and payload size.
- CodeQL (javascript-typescript, security-and-quality) on push/PR + weekly.

## ⚠️ Behaviour notes

- Date-group headers inside the session list and persisted filter/collapse
  selections are session-scoped in this release (the virtualizer needs uniform
  rows; per-feature persistence wiring is staged for a follow-up). Pinned-first
  ordering is preserved.
- The cinematic first-run intro from v1 is not part of the v2 shell.
