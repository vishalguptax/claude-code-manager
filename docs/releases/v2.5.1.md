# v2.5.1

_Released 2026-07-05_

Two fixes to the Account tab and cross-machine setups: the quota card no longer reads as broken when idle, and the SessionStart hook stops duplicating (and triggering Claude Code's broken-command prompt) on machines that sync `~/.claude/settings.json`.

## 🐛 Bug Fixes

- account: reframe quota card so it never reads as broken ([fa72b31](https://github.com/vishalguptax/claude-code-manager/commit/fa72b31b99d6090fcc65a393774c01b53efc8036)) — the 5h/7d numbers only refresh when Claude Code renders its statusline (Anthropic exposes them no other way), so the bars now carry a freshness caption ("Updated 2m ago", or "Updated 40m ago · refreshes when Claude Code runs" once stale) and the button spins on click — the card always explains its state instead of looking like a dead button.
- sessions: dedupe SessionStart hook across synced machines ([83e2868](https://github.com/vishalguptax/claude-code-manager/commit/83e2868cc6bcdbbf64a725586930c246b4855dde)) — the hook installer matched its own entry by exact machine path, so a copy synced from another machine (different home/node path) was left in place and a second entry appended, and Claude Code flagged the now-broken foreign command on every launch. It now matches by script basename, so a foreign copy is recognized and replaced, never duplicated.

## 🔧 Chores
