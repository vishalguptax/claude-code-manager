# v1.3.0

_Released 2026-04-14_

Cross-machine session import/export, a deep audit of session-list bugs and edge cases, and a polish pass over the Account tab.

## ✨ Features

- **Import and export portable session files** — right-click any session → Export Session…, then on the destination machine click the new Import button in the toolbar. Pick the file, choose a target project, confirm, and Claude opens in a fresh terminal with the conversation already loaded. Verified empirically against `claude --resume` (not guessed). Last-folder memory survives across workspaces. ([44dd726](https://github.com/vishalguptax/claude-code-manager/commit/44dd72683f0a5af3e64d1ab3eb8e29b7d1c9b1f6))

## 🐛 Bug Fixes

- **Workspace race on cold start** — sessions list could show `0` because `getWorkspace()` returned `""` before VS Code resolved its workspace folders. Fixed via re-sync on `onDidChangeWorkspaceFolders` plus a belt-and-suspenders re-post on every reload. ([984354b](https://github.com/vishalguptax/claude-code-manager/commit/984354b))
- **Schema drift detection** — parser used to silently drop every session if the Claude CLI renamed required fields. Now surfaces a clear webview error when more than 80% of entries are missing required fields. ([984354b](https://github.com/vishalguptax/claude-code-manager/commit/984354b))
- **Symlinked `~/.claude/` broke file watchers** — `fs.realpathSync` resolves the canonical path before passing to `createFileSystemWatcher`, so dotfile-symlinked Claude directories now refresh automatically. ([984354b](https://github.com/vishalguptax/claude-code-manager/commit/984354b))
- **Multi-root workspaces** — no longer blindly use the first folder; now prefers the folder containing the active editor. ([984354b](https://github.com/vishalguptax/claude-code-manager/commit/984354b))
- **Windows path case mismatch** — pre-computed `projectKey` enables case-insensitive matching so sessions appear regardless of drive-letter casing. ([984354b](https://github.com/vishalguptax/claude-code-manager/commit/984354b))
- **Filter state not persisted** — wired into `vscode.setState` so the user's most recent in-app filter survives panel reloads. ([984354b](https://github.com/vishalguptax/claude-code-manager/commit/984354b))
- **Settings ignored while panel open** — `onDidChangeConfiguration` now re-posts settings to the webview, no reload required. ([984354b](https://github.com/vishalguptax/claude-code-manager/commit/984354b))
- **Watcher posting to disposed webview** — `view = undefined` on dispose, existing null-checks now fire correctly. ([984354b](https://github.com/vishalguptax/claude-code-manager/commit/984354b))
- **Empty-state when current project is unknown** — gracefully shows all sessions instead of zero, so the cold-start window never looks like the bug "no sessions yet" message. ([984354b](https://github.com/vishalguptax/claude-code-manager/commit/984354b))
- **Stuck hover, dropdowns, and "Copied!" timers across the whole panel** — VS Code webview iframes don't fire `mouseleave` when the cursor exits, so visual state used to stick on the last touched element. A global blur + `visibilitychange` handler now force-recomputes `:hover` and clears transient overlays. ([984354b](https://github.com/vishalguptax/claude-code-manager/commit/984354b))

## ⚡ Performance

- **Pre-computed search keys** — `searchHaystack` is built once per session at parse time so search-as-you-type runs one `includes()` per session instead of four `.toLowerCase().includes()` calls. ~7 ms → ~1 ms per keystroke at 350 sessions; ~100 ms → ~12 ms at 5,000 sessions. ([984354b](https://github.com/vishalguptax/claude-code-manager/commit/984354b))
- **Project filter dropdown** — replaced O(P × N) per-project filter calls with a single O(N) count map built in one pass. Eliminates ~250K filter ops per dropdown open at 5K sessions. ([984354b](https://github.com/vishalguptax/claude-code-manager/commit/984354b))

## 💅 UI / Styling

- **Voice dictation toggle** — replaced the raw native checkbox with a compact pill switch. Off-white thumb (never pure `#fff`) so it never blows out against the accent track. Theme-aware via VS Code button background. ([c262f54](https://github.com/vishalguptax/claude-code-manager/commit/c262f54))
- **Model dropdown chevron** — stripped the inconsistent browser-default arrow, replaced with a Lucide chevron rendered via the existing icon system. Matches every other chevron in the panel. ([c262f54](https://github.com/vishalguptax/claude-code-manager/commit/c262f54))
- **Model options** — reordered to match Claude CLI `/model` output (Default → Sonnet → Haiku → Opus) with short purpose-driven helper-line descriptions. Default explicitly names the locked Opus 4.6 + 1M context combo. ([c262f54](https://github.com/vishalguptax/claude-code-manager/commit/c262f54))
- **Heatmap opacity ramp** — replaced the broken 0.4 / 0.2 / 0.45 / 0.7 / 1.0 steps (where a no-activity day could look brighter than a 1-message day on dark themes) with a clean monotonic 0.08 / 0.30 / 0.55 / 0.80 / 1.0 ramp. ([c262f54](https://github.com/vishalguptax/claude-code-manager/commit/c262f54))
- **Stats numbers** — bumped from `--fs-lg` to `--fs-xl` with tighter line-height so metrics dominate the cards instead of reading as body text. ([c262f54](https://github.com/vishalguptax/claude-code-manager/commit/c262f54))
