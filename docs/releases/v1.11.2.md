# v1.11.2

_Released 2026-05-28_

The 1.11.0 statusline-based quota silently failed in any repo whose own `.claude/settings.json` defined a statusline — Claude Code's precedence is local › project › global, so a project statusline shadowed our global tap. This release makes the install scope-aware: the tap goes into the scope Claude actually reads, and uninstall reverses it at the same scope. Adds a live cache watcher so the panel updates the instant Claude re-renders (no manual refresh) and tightens a few details surfaced during testing.

## ✨ Features

- account: install the statusline tap at the **effective** scope (local › project › global) so a project/local statusline doesn't shadow it; uninstall reverses at the recorded scope ([d4e0ad6](https://github.com/vishalguptax/claude-code-manager/commit))
- account: live cache watcher — the Quota + Current Session cards auto-refresh the moment Claude Code re-renders its statusline, no manual click ([d4e0ad6](https://github.com/vishalguptax/claude-code-manager/commit))

## 🐛 Bug Fixes

- account: bake an absolute `node` path at install time (resolved via `which`/`where`) so the command works under nvm and other setups where VS Code's PATH differs from the shell's
- account: re-installs / scope changes now clean orphan tap commands out of non-effective scopes — the tap lives in exactly one place at any time
- account: misleading "resets now" replaced with "outdated · open Claude to refresh" when a cached window has already rolled over (it was past, not imminent)
