# v2.8.0

_Released 2026-07-23_

Sessions now understand git worktrees — the sidebar shows which worktree a session ran in, groups every worktree of a repo together, and resumes into the right checkout instead of forcing a branch switch.

## ✨ Features

- **git worktree awareness — detect, group, resume, recreate** ([98c839c](https://github.com/vishalguptax/claude-code-manager/commit/98c839ca4178702e95745eddf7f0072b152f6d14))
  - Detects both Claude-created worktrees (`.claude/worktrees/`, `worktree-*` / `pr-N` branches) and user-created ones (`git worktree add`), tagging each session with a worktree badge.
  - Groups every worktree of a repository under one entry instead of scattering them across separate "projects", and adds a checkout filter (Main / Claude / User).
  - Worktree-aware resume: a session whose worktree still exists resumes in place with no `git checkout` — fixing the failure that happened when the branch was already checked out in another worktree. When a branch lives in a sibling worktree, offers to open it.
  - Recreate a removed Claude worktree (`git worktree add`) behind a confirm modal, then resume the session there.
