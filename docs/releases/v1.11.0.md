# v1.11.0

_Released 2026-05-28_

The Quota card no longer calls Anthropic's OAuth usage endpoint. Earlier versions sent your subscription OAuth token to `api.anthropic.com` from outside Claude Code — which Anthropic's 2026 terms now prohibit and actively enforce, putting accounts at risk. Quota is now read **entirely locally**: an opt-in statusline tap lets Claude Code (the authorized client) fetch the figures and cache them, and the panel just reads that file. No network call, no token. A new **Current session** card surfaces the live model, context-window usage, and session cost from the same cache.

## ✨ Features

- account: read 5h / 7d quota from a local statusline cache — no network call, no OAuth token ([17eeac6](https://github.com/vishalguptax/claude-code-manager/commit/17eeac63c895d711ff53ee8d46542e0883eee8c3))
- account: new Current session card — live model, context usage, and session cost ([17eeac6](https://github.com/vishalguptax/claude-code-manager/commit/17eeac63c895d711ff53ee8d46542e0883eee8c3))
- account: opt-in "Enable live quota" wires a statusline tap that chains your existing statusline and is fully reversible ([17eeac6](https://github.com/vishalguptax/claude-code-manager/commit/17eeac63c895d711ff53ee8d46542e0883eee8c3))

## 🔒 Compliance

- Removes the OAuth `/usage` request that used the subscription token outside Claude Code — a Consumer Terms §3 violation under Anthropic's January–April 2026 enforcement. The extension is now fully local for quota.

## Notes

- "Refresh" re-reads the local cache, so the figures are as fresh as Claude Code's last statusline render (continuously updated while a session is active). It never forces a server request.
- The per-model 7-day and pay-as-you-go rows are gone — the statusline payload doesn't carry them.
