# Product

## Register

brand

## Users

Developers who already use Claude Code (the CLI, the official VS Code extension, or both) and work inside a VS Code-based editor (VS Code, Cursor, Windsurf, Antigravity, VSCodium, Codespaces, Gitpod). They arrive from a Google search ("claude code session manager", "claude account switcher vscode", "claude usage tracker"), a marketplace listing, or a shared link on Reddit/X. Context: evaluating whether to install a tool that touches their local `~/.claude/` data, so trust and privacy are decision factors, not afterthoughts. The job to be done: decide in under a minute that this is safe, useful, and worth installing, then click through to the Marketplace or Open VSX.

## Product Purpose

The site at claudemanager.vishalg.in is the marketing and discovery surface for the Claude Manager VS Code extension. It exists to (1) rank on Google for Claude Code tooling queries the Marketplace README cannot win, and (2) convert that traffic into installs. Success = first-page ranking for target long-tail queries plus a measurable install lift attributable to the site. It is not documentation; deep reference stays in the repo README.

## Brand Personality

Technical and trustworthy. Developer-credible, precise, calm. Three words: precise, local, honest. The voice states what the extension literally does (browse sessions, switch accounts, track usage, edit MCP without JSON) and backs the privacy claim plainly ("100% local, zero telemetry") rather than selling it. Confidence comes from specificity, not adjectives. Emotional goal: a developer feels this was built by someone who uses Claude Code daily and respects their machine.

## Anti-references

- **Generic SaaS template**: cream/sand body background, gradient-text hero, identical icon-card grids repeated endlessly, tiny uppercase tracked eyebrows above every section. The AI-slop default.
- **Crypto/AI hype**: neon glows, dark-with-purple-gradient drench, buzzwords (supercharge / unleash / next-generation), fake urgency or countdowns.
- **Over-animated**: scroll-jacking, uniform fade-in-on-every-section, parallax for its own sake. Motion must mean something or stay off.

## Design Principles

- **Practice what you preach.** The site is fast, local-friendly, and free of trackers, mirroring the product's privacy promise. No analytics scripts that contradict "zero telemetry".
- **Show the real thing.** Use actual extension screenshots and the real demo, not illustrations or mockups. Specificity is the credibility.
- **Specific noun, plain verb.** Copy names the literal capability ("switch Claude accounts without logging out") over abstract benefit language.
- **Earn every element.** No decorative section that exists only because landing pages "usually have one". Each block moves a developer toward an informed install decision.
- **Privacy is the headline, not the footnote.** Trust signals (local, zero telemetry, open source, Apache 2.0) are surfaced early and stated without hedging.

## Accessibility & Inclusion

WCAG 2.1 AA. Body text contrast >=4.5:1 and large text >=3:1 in the dark theme; visible focus rings; keyboard-navigable; descriptive alt text on every screenshot and the demo; `prefers-reduced-motion` honored with transitions disabled. Color is never the sole carrier of meaning.
