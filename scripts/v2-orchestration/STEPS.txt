  PERMISSIONS NOTE
================================================================

Every Claude session below uses:

multi-hour sessions in isolated worktrees.

Risks:
  - Claude has full tool access (Bash, Write, etc.)
  - Each session sandboxed to own worktree by file allowlist in prompt

You will need to babysit each session and approve every tool call.
For 8 parallel sessions, this is impractical. Flag recommended.

================================================================
  TROUBLESHOOTING
================================================================

Claude session goes off the rails:
    Type in same session: "Stop. Re-read prompt file. Restart from step X."
    Do NOT exit and restart Claude - lose context.

Build fails after a phase:
    In phase's claude session: "npm run build fails with: <error>. Fix it."
    Claude fixes in place. Re-verify, then merge.

Merge conflict:
    cd <integration-target worktree>
    git merge --abort
    Inspect conflicting branch. Decide: re-run that F2 session, or
    manually resolve. For manual:
        git merge --no-ff <branch>
        # Resolve conflicts in editor
        git add .
        git commit
        git push

Worktree stuck:
    git worktree remove <path> --force
    git worktree prune
    Re-create via setup-worktrees.ps1

Claude CLI session token expired:
    Re-run: claude login
    Then re-launch session.

Lost track of which session is which:
    Window title or run: git branch --show-current
    Each worktree on dedicated branch.

Budget exhausted in a session:
    Claude warns before stopping. Topup credits, then "continue" in
    same session.

Out of disk space (12 worktrees take ~2 GB):
    Remove finished worktrees as soon as their branch is merged.
